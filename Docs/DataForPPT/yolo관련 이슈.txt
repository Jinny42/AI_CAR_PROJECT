	     아무것도 인지하지 못했을 경우, 아무 메세지를 보내지 않아 프로그램의

		 반응이 느려짐

 

solve 1. background 상에 ros_timer를 추가 후, 아무 물체도 인식하지 못한 채로

		 카운트를 세고, 카운트가 기준 이상을 넘을 경우, 아무것도 인식하지 못

		 함에 따른 메세지 발송.

		 물체가 인식 되었다면, 카운트를 다시 0으로 돌려줌.

 

issue 2. 수정 전 처리에서는 '0'이나 '47.0000000', 'inf' 라는 값을 거르지 않

		 고 그에 따른 명령처리를 한 if-else if문에서 같이하고 그에 따른 메세

		 지를 보냄. 그럴 경우, 걸러야 할 값들이 한번씩 튀어나와 Lidar의 값이

		 불안정해지고, 차량이 예상했던 결과와 다르게 버벅거리는 문제 발생

 

solve 2. 수정 후에는 if-else if문을 이중 if문으로 바꾸어, 걸러야 할 값들을

		 거른 후, Lidar값에 따른 명령 메세지를 보내토록 함. 버벅거리는 문제

		 해결.

 

issue 3. object detection 과정에서 차와 신호등을 각각 다른 노드에서 같은

		 메세지 파일을 이용해 그에 따른 명령을 수행토록 함. 이 과정에서 차를

		 인식했을 경우와 신호등을 인식했을 경우, 두 명령 중 하나만 수행이 됨

 

solve 3. 메세지 파일을 하나 더 만든 후, 각각 다른 노드에서 다른 메세지 파일

		 을 이용하도록 함. 두 명령 중 하나만 수행하던 문제가 해결되고, 두 

		 명령 동시에 수행이 가능해짐.
